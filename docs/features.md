# Описание экранов и функций

## Главный экран (`HomeScreen`)

**Маршрут:** `/`

- Баннеры акций (`PromoBanner`) — горизонтальный список из таблицы `promotions`
- Популярные блюда (`is_popular = true`) — горизонтальный скролл
- Сетка категорий — переход на `/category/:id`

Данные загружаются параллельно через `popularItemsProvider` и `promotionsProvider`.

---

## Экран категории (`CategoryScreen`)

**Маршрут:** `/category/:id?name=Плов`

- Список блюд выбранной категории (`menuItemsByCategoryProvider`)
- Фильтрует только `is_available = true`
- Каждая карточка (`MenuItemCard`): фото, название, вес, цена, кнопка "+ в корзину"
- Tap по карточке → `/item/:id` (передаётся объект `MenuItem` через `extra`)

---

## Детальная страница блюда (`ItemDetailScreen`)

**Маршрут:** `/item/:id` (объект `MenuItem` в `extra`)

- Большое изображение (hero-анимация)
- Описание, вес, бейджи `Халяль` / `Острое`
- Кнопка "Добавить в корзину" → добавляет через `cartProvider`

---

## Корзина (`CartScreen`)

**Маршрут:** `/cart`  
**Доступ:** FAB на главной навигации (показывает количество позиций)

- Список позиций с управлением количеством (`+` / `-` / удалить)
- Итоговая сумма
- Кнопка "Оформить заказ" → `/checkout`
- При пустой корзине — пустой стейт с кнопкой "В меню"

**Состояние хранится в памяти** (`CartNotifier`): сбрасывается при перезапуске приложения.

## Оформление заказа (`CheckoutScreen`)

**Маршрут:** `/checkout`

### Поля формы

| Поле | Обязательное | Описание |
|---|---|---|
| Тип получения | Да | Доставка / Самовывоз |
| Адрес | Если доставка | Улица, дом, квартира |
| Имя | Нет | Автозаполнение из профиля |
| Телефон | Да | Префикс +7, 10 цифр |
| Комментарий | Нет | Пожелания, аллергии |

### Логика отправки

1. Если пользователь не авторизован — редирект на `/auth`
2. Запись заказа в таблицу `orders` (Supabase)
3. Открытие WhatsApp с готовым сообщением на номер `+7 707 400 77 28`
4. Очистка корзины → переход на `/order-sent`

Если запись в Supabase упала (нет сети) — заказ всё равно уходит в WhatsApp.

### Формат WhatsApp-сообщения

📋 Состав:
- Плов классический x2 — 5 000 тг
- Шашлык из баранины x1 — 4 500 тг

💰 Итого: 9 500 тг
🚗 Доставка
💵 Оплата: Наличные / Kaspi перевод
📍 Адрес: ул. Абая 10, кв 5
👤 Имя: Асан
📞 Телефон: +77771234567

⏰ 14:32

## Авторизация (`AuthScreen`)

**Маршрут:** `/auth`  
**Когда показывается:** при попытке оформить заказ без авторизации

### Шаг 1 — ввод телефона
- Поле: +7 + 10 цифр
- Кнопка "Получить код" → `Supabase.auth.signInWithOtp(phone: ...)`

### Шаг 2 — ввод OTP
- 6-значный код из SMS
- `Supabase.auth.verifyOTP(phone, token, type: OtpType.sms)`
- После успеха — `context.pop()` (возврат на checkout)
- Кнопка "Изменить номер" — возврат на шаг 1

Требует настроенного SMS-провайдера в Supabase. См. [supabase-setup.md](supabase-setup.md#5-настроить-phone-auth-otp).

## Бронирование стола (`ReservationScreen`)

**Маршрут:** `/reservation`  
**Доступ:** вкладка в BottomNavigationBar

### Поля формы

| Поле | Значение по умолчанию | Ограничения |
|---|---|---|
| Дата | Завтра | min: сегодня, max: +60 дней |
| Время | 13:00 | Пикер системного времени |
| Количество гостей | 2 | min: 1, max: 20 |
| Имя | — | Обязательное |
| Телефон | — | Обязательное |
| Комментарий | — | Необязательное |

### Логика отправки

1. Запись в таблицу `reservations` (не требует авторизации — гость может бронировать)
2. Открытие WhatsApp с сообщением о брони
3. SnackBar "Запрос на бронирование отправлен!"

### Формат WhatsApp-сообщения

```
🍽 Бронирование стола — ПЛОВ НОМЕР 1

👤 Имя: Асан
📞 Телефон: +77771234567
📅 Дата: 20.05.2026
⏰ Время: 13:00
👥 Гостей: 4
💬 Отдельный зал
```
